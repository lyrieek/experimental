报错原因

@Autowired(required=true)
private ILoginService loginService;

把

required=true

改为

required=false

启动就不报错了，但是service就没注入，所以在用到loginService就会报空指针异常，其实主要报这个错的原因是因为spring配置文件中的启动注解配置包名的时候没配置到service包，我的整个spring MVC配置都是百度的，自己不会配，所以就有了下面的配置

<context:component-scan base-package="com.xxxx.controller" />

结果就报了上面的错，改为

<context:component-scan base-package="com.xxxx.*" />

就ok了
一般@Autowired注入失败有以下一个原因：
1.包没扫描到，就是上面的。
2.controller层的@controller或者service层的@Service注解没写
3.如果使用@Resource 注入service需要注意对象名字,如:

@Service("aService")
public class AServiceImpl extends BaseService implements IAService{}

那么用@Resource注入IAService时对象名需要和service里面的名字一样;

@Resource
private IAService aService

如果不一样可能会报bean不存在的错
